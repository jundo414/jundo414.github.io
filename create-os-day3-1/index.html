ブートセクタの実装

きちんとしたIPLの作成

今回は、前回の “MOV DS,AX” 以降に以下の項目を追加する。

JC はキャリーフラグが 1 の場合に遷移する命令。
キャリーフラグ (CF) は、符号なし演算でオーバーフローして正常な値を返せない場合にフラグがセットされる。
類似のものにオーバーフローフラグ (OF) が存在するが、これは、符号あり演算の時に使用され、以下の場合にフラグがセットされる。

  • 負の値 + 負の値 = 正の値
  • 正の値 + 正の値 = 負の値

「INT 0x13」については、以下の (AT)BIOS のページによると以下の記載がある。

  • http://oswiki.osask.jp/?(AT)BIOS

フロッピーディスクの構成は以下の通り。（書籍より拝借）

ここでは、フロッピーの図を掲載したが、HDD とかでもほぼ同じ。

要約すると以下の処理を行なっている。

  • AH = ディスクからの読み込み
  • AL = 処理するセクタ数 = 1
  • CH = シリンダ番号(0-79) = 0
  • CL = セクタ番号(1-18) = 2
  • DH = ヘッド番号（表側か裏側か） = 0
  • DL = ドライブ番号（何番目のドライブか） = 0x00
  • ES:BX = バッファアドレス(最大で1,114,095バイト) = ES 16 + BX = 0x0820 16 + 0 = 0x8200番地〜0x83ff番地

メモリの番地指定でもセグメントレジスタの指定が必要で、未指定の場合、DS: が設定される。
そのため、初期化処理で、DS に 0 をセットしている。

読み込みエラー時のリトライを追加

ここでは、retry セクタを追加しエラーが発生した際に最大5回までリトライするようにしている。

JNC は、キャリーフラグが 0、つまり符号なし演算でオーバーフローが発生していなければ遷移する命令。

エラー発生時のリトライ前の初期化処理は、以下の記述で対応している。

(AT)BIOS のページを確認するとシステムのリセット処理を行なっていることがわかる。

18セクタ分を連続で読み込む

今度は、シリンダ番号 0 をセクタ単位でループしながらすべて読み込み、0x0820 番地以降に格納している。

JBE 命令は、”Jump if Below or Equal” の略で、直前で比較した値以下の場合ジャンプするというもの。

処理するセクタ数 (AL) を 17 とせず、あえてループ処理をした理由は、以下の (AT)BIOS ページにある以下の注意書きに対応した結果とのこと。

  • 処理するセクタ数は0x01～0xffの範囲で指定 （0x02以上を指定するときは、連続処理できる条件があるかもしれないので注意 – FDの場合は、たぶん、複数のトラックにはまたがれないし、64KB境界をこえてもいけない、だと思います）

「今回は、AL を 17 にセットしても同様の結果を得られるけど、今後問題になるから、事前にループ処理にしておいた」とのことなので、今後、複数トラックに対して処理したり、64KB 以上のデータを処理したりするのでしょう..きっと..

10シリンダ分を連続で読み込む

JB は “Jump if Below” の略で、比較対象の数値より小さい場合ジャンプする。

上記のコーディングにより、180KBバイトが書き込まれる。

  • シリンダ数(CYLS) x ヘッド数(DH) x セクタ数(CL) x 1セクタ当たりのバイト数 = 10 x 2 x 18 x 512 = 184,320バイト = 180KB
