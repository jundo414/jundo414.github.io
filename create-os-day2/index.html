アセンブラによる OS イメージの作成（その3）

Day2 では、./projects/02_day/helloos3/helloos.asm をベースに作業していく。
「アセンブラによる OS イメージの作成（その２）」よりもさらに人に優しいコードとなっている。
その代わりレジスタや新しいアセンブラ命令が増えているため、新出の命令について以下に簡単に解説しておく。

レジスタ

レジスタとはCPUの記憶回路。

16ビットレジスタ

レジスタ 正式名称
AX Accumulator
CX Counter
DX Data
BX Base
SP Stack Pointer
BP Base Pointer
SI Source Index
DI Destination Index

8ビットレジスタ

レジスタ 正式名称
AL Accumulator Low
CL Counter Low
DL Data Low
BL Base Low
AH Accumulator High
CH Counter High
DH Data High
BH Base High

32ビットレジスタ

レジスタ 正式名称
EAX Extend Accumulator
ECX Extend Counter
EDX Extend Data
EBX Extend Base
ESP Extend Stack Pointer
EBP Extend Base Pointer
ESI Extend Source Index
EDI Extend Destination Index

セグメントレジスタ（16ビット）

レジスタ 正式名称 備考
ES Extra Segment おまけセグメント
CS Code Segment
SS Stack Segment
DS Data Segment
FS - おまけセグメント
GS - おまけセグメント

アセンブラ命令

ORG <メモリアドレス>

以降の命令を<メモリアドレス>以降にセットする。

使用例：

JMP <ラベル>

無条件に指定した<ラベル>に移動する。

使用例：

MOV <オペランド1>, <オペランド2>

<オペランド1>に<オペランド2>を代入する。

使用例：

ここで “[ ]” という表現が出てくるが、これはメモリであることを示している。
また、ビット数の同じもの同士でしか代入できない原則があるため、
“[SI]” は、「メモリの SI 番地の1バイトの内容を AL に代入」を表している。

ADD <オペランド1> <オペランド2>

<オペランド1>に<オペランド2>を加算する。

CMP <オペランド1> <オペランド2>

<オペランド1>と<オペランド2>を比較する。
使用例：

JE <ラベル>

前の比較結果が等しかった場合は指定した<ラベル>へ移動する。
使用例：

INT <オペランド>
